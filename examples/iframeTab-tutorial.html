iframeTab 文檔

引入 iframeTab

本插件是以Gentelella為框架開發的，本demo對此框架有一定依賴性，但實際使用上并非必要。

<!DOCTYPE html>
<html>
<header>
    <meta charset="utf-8">

    <!-- 引入 iframeTab 樣式文件，非必要，可自行定制 -->
    <link href="iframeTab.min.css" rel="stylesheet">

    <!-- 引入 jQuery 和 iframeTab 文件 -->
    <script src="jquery.min.js"></script>
    <script src="iframeTab.min.js"></script>
</header>
</html>

鏈接Dom

<a href="iframeTab-demo.html" data-num="0">主頁</a>

屬性 類型 默認值 描述
data-num Number 0 必要。以此區別一般瀏覽器新開窗口與以iframeTab開啟的鏈接。
data-name String null 可選屬性，默認a標籤內文字，優先級：data-name > a.text() 。
data-mul String null 可選屬性，值不作限制。若存在，則此鏈接可同時多個創建窗口。
data-reload String null 可選屬性，值不作限制。若存在，則再次點擊此鏈接時，iframe將被重新加載。

標籤欄結構

<div id="tabHeader">
    <ul>
        <li class="active tab-keep" data-tab="my-desktop.html" data-num="0">首頁</li>
        <li data-tab="tabPage.html">標籤頁 <i data-btn="close"></i></li>
    </ul>
</div>

<li> 中的 data-tab 應與對應的iframe頁 <iframe> 中的 src 和 data-iframe 保持一致，內容應為iframe的 src 屬性。同時也必須包含 data-num="0" 。
li.active 表示當前標籤已激活，li.tab-keep 表示右鍵刪除所有標籤時將保留當前標籤。
若需添加 data-btn="close" 為刪除標籤按鈕。

iframe結構

<div id="tabBody">
    <div class="tab-panel tab-keep active">
        <!-- 包裹iFrame的外部元素，可按自己需求更改，如需設置，應同時在option中配置iframeBox -->
        <div class="right_col" role="main">
            <iframe src="my-desktop.html" data-iframe="my-desktop.html" data-num="0" marginheight="0" marginwidth="0" frameborder="0" scrolling="no" onload="iframeTab.iframeHeight()" height="188"></iframe>
        </div>
    </div>
</div>

<iframe> 中的 src 和 data-iframe 應與對應的標籤 <li> 中的 data-tab 保持一致，內容應為iframe的 src 屬性。同時也必須包含 data-num="0" 。
.tab-panel.active 表示當前iframe已激活，.tab-panel.tab-keep 表示右鍵刪除所有標籤時將保留當前iframe頁。
<div class="right_col" role="main"> 為 div.tab-panel 和 <iframe> 間的容器，並非必要，若需自行配置或不需要該容器，可在option中配置 iframeBox 參數。

初始化 iframeTab

var tab = iframeTab.init();

通過 iframeTab.init 方法即可初始化iframeTab並返回一個API實例，可根據需求修改配置數據。

var iframeTabOption = {
    closesBtnClass: 'fa fa-close',
    callback: {
        onChange: function () {
            alert('標籤切換了')
        }
    }
}

var tab = iframeTab.init(iframeTabOption);

配置項

  • tabLiClass | String

    標籤欄標籤 <li> 樣式，[ default: '' ]

  • tabPanClass | String

    標籤頁外層容器 <div class="tab-panel"> 樣式，[ default: '' ]

  • closesBtnClass | String

    刪除按鈕 [data-btn="close"] 樣式，[ default: 'fa fa-close' ]

  • switchBtnDown | String

    折疊按鈕折疊狀態樣式，[ default: 'fa fa-chevron-down' ]

  • switchBtnUp | String

    折疊按鈕展開狀態樣式，[ default: 'fa fa-chevron-up' ]

  • contextmenuClass | String

    右鍵菜單 div.tab-contextmenu 樣式，[ default: 'dropdown open' ]

  • iframeBox | String

    div.tab-panel 和 <iframe> 間的容器，非必要，可配置為 '' ，若Html中有修改，此處應跟隨修改，[ default: '<div class="right_col" role="main"></div>' ]

  • singleLineheight | Number

    標籤欄單行狀態高度，[ default: 92 ]

  • menuWidth | Number

    某些UI框架下，如本demo，菜單欄位於右側，iFrame內容頁並非佔據整個視窗寬度，此情況下應配置菜單欄寬度，[ default: 230 ]

  • callback | Object

    用於的事件回調函數，[ default: null ]

    子參數 類型 默認值 描述
    beforeCreat Function null 用於捕獲窗口切換之前的事件回調函數，並根據返回值確定是否允許切換窗口操作。
    onCreat Function null 用於捕獲窗口創建時的事件回調函數，若設置了callback.beforeCreat方法，且返回false，將無法出發onCreat事件回調函數。
    afterCreat Function null 用於捕獲窗口創建之後的事件回調函數，若設置了callback.beforeCreat方法，且返回false，將無法出發afterCreat事件回調函數。
    beforeChange Function null 用於捕獲窗口切換之前的事件回調函數，並根據返回值確定是否允許切換窗口操作。
    onChange Function null 用於捕獲窗口切換時的事件回調函數，若設置了callback.beforeChange方法，且返回false，將無法出發onChange事件回調函數
    afterChange Function null 用於捕獲窗口切換之後的事件回調函數，若設置了callback.beforeChange方法，且返回false，將無法出發afterChange事件回調函數。
    beforeClose Function null 用於捕獲窗口關閉之前的事件回調函數，並根據返回值確定是否允許關閉窗口操作。
    onClose Function null 用於捕獲窗口關閉時的事件回調函數，若設置了callback.beforeClose方法，且返回false，將無法出發onClose事件回調函數。
    afterClose Function null 用於捕獲窗口關閉之後的事件回調函數，若設置了callback.beforeClose方法，且返回false，將無法出發afterClose事件回調函數。

    實例：禁止操作

    var option = {
        callback: {
            beforeCreat: function () {  // 禁止創建窗口操作
                return false
            },
            beforeChange: function () {  // 禁止切換窗口操作
                return false
            },
            beforeClose: function () {  // 禁止關閉窗口操作
                return false
            }
        }
    }

API

iframeTab.iframeHeight(updateHeight)

刷新iframe窗口高度。在某些場合上如通過JS對頁面進行動態修改，頁面高度改變會出現窗口高度過短，此時可通過此方法刷新頁面高度。

若在子窗口調用，需加上 parent。

parent.iframeTab.iframeHeight(updateHeight)

參數：

  • updateHeight | Number

    某些情況，頁面包含脫離文檔流的元素且超出document高度，此時應獲取該元素高度并作為參數傳入。
    忽略此參數，表示取iframe的document高度。

返回對象的API

var tab = iframeTab.init()

初始化iframeTab時將同時創建一個API實例，返回的對象可使用以下API方法。

tab.on(eventName, handler) | eventName: String, handler: Function
tab.on(map) | Object

綁定事件處理函數，用於監聽並執行你的代碼。此方法將覆蓋初始化iframeTab時傳入的callback參數。

參數：

  • eventName | String

    事件名稱。語法類似 jQuery.on()，可一次傳入一個或多個事件，以空格分割。

  • handler | Function

    事件處理函數。

  • map | Object

    Object對象，每個屬性對應事件名稱，屬性值對應綁定的事件處理函數（參數：handler）。

事件名稱 描述
beforeCreat 此事件在窗口被創建前被觸法。若返回值為 false，將禁止創建窗口操作。在此事件下執行操作被激活的iframe的函數，將操作到舊有被激活窗口上。
onCreat 此事件在iframe加載完畢前被觸法。
afterCreat 窗口被創建且iframe加載成功後觸法此事件，在此事件下執行操作iframe的函數，將操作到新窗口上。
beforeChange 此事件在窗口被切換前被觸法。若返回值為 false，將禁止切換窗口操作。在此事件下執行操作被激活的iframe的函數，將操作到舊有被激活上。
onChange 此事件在舊有iframe被隱藏，且新iframe尚未被顯示時被觸法。
afterChange 窗口被切換後觸法此事件，在此事件下執行操作iframe的函數，將操作到新窗口上。
beforeClose 此事件在窗口被關閉前被觸法。若返回值為 false，將禁止關閉窗口操作。在此事件下執行操作被激活的iframe的函數，將操作到舊有被激活上。
onClose 此事件在舊有iframe被關閉，且新iframe尚未被創建時被觸法。
afterClose 窗口被關閉後觸法此事件，在此事件下執行操作iframe的函數，將操作到新窗口上。

tab.off(eventName) | String

解綁通過 on() 方法綁定的事件和移除通過option設定的事件回調函數。
參數接受String或忽略。

類型 描述
String 傳入一個或多個事件名稱，以空格分割，將解綁該事件或對應的事件回調函數。
忽略 若忽略此參數，將解綁所有事件監聽並移除所有事件回調函數。

tab.closeActIframe()

執行此方法，將關閉當前iframe窗口。

tab.destroy()

銷毀實例，銷毀後超連結將創建窗口到瀏覽器標籤爛上，除 dispose 方法外，所有實例方法也無法再被使用。
重新使用iframeTab，必須使用 init 方法重新初始化。

tab.dispose(refName) | String

解綁所有事件、銷毀實例的同時，釋放對象。

參數：

需傳入聲明保存實例的全局變量名稱，否則將無法釋放。此例需傳入的是 tab。

按鈕

在元素上添加 [data-btn="refresh"] ，則點擊該元素時，元素所在iFrame頁將被刷新。

gulp

因為本插件涉及iframe操作，在本地直接打開將報錯，你需要在服務器上調試，本插件提供 gulp 調試方式，你可以以此進行插件模擬和代碼編譯。

  • gulp 運行服務器并編譯所有代碼
  • gulp browser-sync 運行服務器
  • gulp watch 進入watch模式
  • gulp sass-to-css 將sass編譯成css
  • gulp minify-css 壓縮css
  • gulp jscompress 壓縮js
